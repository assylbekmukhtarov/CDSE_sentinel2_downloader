ЧТО ДЕЛАЕТ СКРИПТ
-----------------
1. Авторизуется на dataspace.copernicus.eu через OAuth 2.0 (Password Grant).
   Токен автоматически обновляется во время работы.

2. Ищет снимки Sentinel-2 уровня L2A (атмосферно скорректированные) по фильтрам:
   - Регион:       bbox Краснодарского края (см. BOUNDS ниже)
   - Период:       01.01.2016 -- 31.12.2025
   - Облачность:   не более 30%
   - Тип продукта: S2MSI2A

3. Обрабатывает по 100 снимков за раз (батч):
   получил 100 -> скачал -> получил следующие 100 -> скачал -> ...
   Всего в каталоге по этим параметрам ~15 000+ продуктов.

4. Для каждого продукта скачивает 12 каналов отдельными JP2-файлами:
   B01  Coastal aerosol  443 nm   60 м/пикс
   B02  Blue             490 nm   10 м/пикс
   B03  Green            560 nm   10 м/пикс
   B04  Red              665 nm   10 м/пикс
   B05  Red Edge 1       705 nm   20 м/пикс
   B06  Red Edge 2       740 nm   20 м/пикс
   B07  Red Edge 3       783 nm   20 м/пикс
   B08  NIR              842 nm   10 м/пикс
   B8A  Narrow NIR       865 nm   20 м/пикс
   B09  Water vapour     945 nm   60 м/пикс
   B11  SWIR 1          1610 nm   20 м/пикс
   B12  SWIR 2          2190 nm   20 м/пикс

5. Если каналы не найдены (редкий случай) -- скачивает полный ZIP продукта.
   При ошибке скачивания ZIP продукт НЕ помечается как готовый — будет повтор.

6. Поддерживает возобновление после прерывания (Ctrl+C):
   - done_products.txt  -- список уже скачанных продуктов
   - last_skip.txt      -- позиция в каталоге (номер батча)
   Повторный запуск продолжит с места остановки.

7. Прогресс скачивания каждого файла отображается через tqdm в терминале.
   В лог-файл прогресс не пишется — это нормально.


ФОРМАТ ФАЙЛОВ JP2
-----------------
Sentinel-2 распространяет данные в формате JPEG 2000 (.jp2).
Каждый канал -- отдельный файл, один файл = одна полутоновая растровая сетка.
Для работы с JP2 подходят: QGIS, GDAL, rasterio, Snap.

Если нужен один многоканальный GeoTIFF -- стекировать через gdal_merge или rasterio.

КАК РАБОТАЕТ NODES API
-----------------------
Скрипт использует Nodes API на download.dataspace.copernicus.eu для навигации
по внутренней структуре продукта .SAFE и получения прямых ссылок на JP2-файлы.

Путь до JP2 внутри продукта:
  .SAFE -> GRANULE -> <тайл> -> IMG_DATA -> R<разрешение>m -> <канал>.jp2

URL для скачивания строится автоматически из uri-поля каждого узла.
Ответ API использует ключ 'result' (не 'value').

НАСТРОЙКИ В СКРИПТЕ (download_sentinel.py)
------------------------------------------
  MAX_CLOUD        = 30    -- максимальная облачность, %
  START_DATE       = 2016-01-01
  END_DATE         = 2025-12-31
  PAGE_SIZE        = 100   -- продуктов за один запрос к каталогу
  RETRY_LIMIT      = 5     -- попыток при ошибке сети
  RETRY_DELAY      = 10    -- секунд между попытками
  REQUEST_TIMEOUT  = 120   -- секунд ожидания ответа сервера
  NODES_BASE       = https://download.dataspace.copernicus.eu/odata/v1

ПЕРВЫЙ ЗАПУСК: ПОШАГОВАЯ ИНСТРУКЦИЯ
------------------------------------
1. Зарегистрируйтесь на https://dataspace.copernicus.eu (бесплатно).

2. Установите зависимости:
     pip install requests python-dotenv tqdm

3. Заполните .env:
     CDSE_USER=ваш_email@example.com
     CDSE_PASS=ваш_пароль

4. Запустите скрипт:
     python download_sentinel.py

5. Для остановки нажмите Ctrl+C.
   Следующий запуск продолжит с места остановки.


ВОЗМОЖНЫЕ ОШИБКИ
----------------
  KeyError: CDSE_USER
      -- не найден .env файл или переменная не задана

  401 Unauthorized
      -- неверный логин/пароль в .env

  Nodes API: каналы не найдены
      -- структура продукта нестандартная, скачивается полный ZIP (~600 МБ)

  Ошибка скачивания ZIP
      -- продукт не помечается как готовый, повторится при следующем запуске

  tqdm не отображается в терминале
      -- tqdm пишет в stderr; прогресс виден в терминале, но не в download.log
      -- файлы .tmp в папке продукта означают что скачивание идёт прямо сейчас
